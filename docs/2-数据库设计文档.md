### 关系说明

1. **User ─┬─ CartItem**：一对多关系
   - 一个用户可以有多个购物车项
   - 级联操作：删除用户时删除其购物车

2. **User ─┬─ Order**：一对多关系
   - 一个用户可以有多个订单
   - 保留历史：删除用户时保留订单记录

3. **CartItem ─── Product**：多对一关系
   - 多个购物车项可以指向同一商品
   - 删除商品时删除关联购物车项

4. **Order ◄─►  Product**：多对多关系
   - 通过`order_products`中间表实现
   - 一个订单包含多个商品
   - 一个商品可以在多个订单中

## 二、数据库表结构说明

### 表1: users (用户表)

**表说明**：存储用户账号信息，系统核心表

| 字段名 | 数据类型 | 长度 | 是否为空 | 是否主键 | 默认值 | 注释 |
|--------|---------|------|---------|---------|-------|------|
| id | BIGINT | - | NO | PK | AUTO_INCREMENT | 用户ID，自增主键 |
| username | VARCHAR | 50 | NO | - | - | 用户名，唯一 |
| password | VARCHAR | 255 | NO | - | - | 密码，BCrypt加密 |
| email | VARCHAR | 100 | NO | - | - | 邮箱，唯一 |
| created_at | DATETIME | - | YES | - | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | YES | - | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY: `id`
- UNIQUE INDEX: `username`
- UNIQUE INDEX: `email`

**约束**：
- `username` 必须唯一，长度3-50字符
- `email` 必须唯一，符合邮箱格式
- `password` 存储BCrypt加密后的哈希值（60字符）

**示例数据**：
```sql
INSERT INTO users (username, password, email) VALUES
('zhaoyingchen33', '$2a$10$...', 'mc3184046135@outlook.com'),
('testuser', '$2a$10$...', 'test@example.com'),
('admin', '$2a$10$...', 'admin@example.com');
```

---

### 表2: products (商品表)

**表说明**：存储游戏商品信息，包含130+款游戏

| 字段名 | 数据类型 | 长度 | 是否为空 | 是否主键 | 默认值 | 注释 |
|--------|---------|------|---------|---------|-------|------|
| id | BIGINT | - | NO | PK | AUTO_INCREMENT | 商品ID |
| name | VARCHAR | 200 | NO | - | - | 游戏名称 |
| description | TEXT | - | YES | - | - | 游戏描述 |
| price | DECIMAL | 10,2 | NO | - | 0.00 | 原价（美元） |
| discount_percent | INT | - | NO | - | 0 | 折扣百分比（0-100） |
| header_image_url | VARCHAR | 500 | YES | - | - | 游戏封面图URL |
| tags | VARCHAR | 200 | YES | - | - | 游戏标签（逗号分隔） |
| review_summary | VARCHAR | 100 | YES | - | - | 用户评价摘要 |
| developer | VARCHAR | 100 | YES | - | - | 开发商 |
| publisher | VARCHAR | 100 | YES | - | - | 发行商 |
| featured | BOOLEAN | - | NO | - | FALSE | 是否精选 |

**索引**：
- PRIMARY KEY: `id`
- INDEX: `name` (用于搜索)
- INDEX: `featured` (用于精选查询)

**约束**：
- `price` >= 0
- `discount_percent` 范围 0-100
- `featured` 默认为 false

**示例数据**：
```sql
INSERT INTO products (name, description, price, discount_percent, header_image_url, tags, review_summary, developer, publisher, featured) VALUES
('Elden Ring', 'THE NEW FANTASY ACTION RPG...', 59.99, 0, 'https://...', 'RPG, Open World, Dark Fantasy', 'Overwhelmingly Positive', 'FromSoftware', 'Bandai Namco', 1),
('Cyberpunk 2077', 'Cyberpunk 2077 is an open-world...', 59.99, 50, 'https://...', 'RPG, Sci-fi, Open World', 'Very Positive', 'CD PROJEKT RED', 'CD PROJEKT RED', 1);
```

**计算字段**：
- **最终价格** = `price * (1 - discount_percent / 100.0)`
- **是否免费** = `price == 0`

### 表3: cart_items (购物车表)

**表说明**：存储用户购物车内容，临时数据

| 字段名 | 数据类型 | 长度 | 是否为空 | 是否主键 | 默认值 | 注释 |
|--------|---------|------|---------|---------|-------|------|
| id | BIGINT | - | NO | PK | AUTO_INCREMENT | 购物车项ID |
| user_id | BIGINT | - | NO | FK | - | 用户ID，外键 |
| product_id | BIGINT | - | NO | FK | - | 商品ID，外键 |

**索引**：
- PRIMARY KEY: `id`
- UNIQUE INDEX: `(user_id, product_id)` - 防止同一用户重复添加同一商品
- FOREIGN KEY: `user_id` REFERENCES `users(id)` ON DELETE CASCADE
- FOREIGN KEY: `product_id` REFERENCES `products(id)` ON DELETE CASCADE

**约束**：
- **唯一约束**：同一用户不能重复添加同一商品
- **级联删除**：删除用户时自动删除其购物车
- **级联删除**：删除商品时自动删除关联购物车项

**示例数据**：
```sql
INSERT INTO cart_items (user_id, product_id) VALUES
(1, 5),  -- 用户1的购物车中有商品5
(1, 12), -- 用户1的购物车中有商品12
(2, 3);  -- 用户2的购物车中有商品3
```

**业务规则**：
- 结账后购物车自动清空
- 已拥有的游戏不能加入购物车

---

### 表4: orders (订单表)

**表说明**：存储订单基本信息

| 字段名 | 数据类型 | 长度 | 是否为空 | 是否主键 | 默认值 | 注释 |
|--------|---------|------|---------|---------|-------|------|
| id | BIGINT | - | NO | PK | AUTO_INCREMENT | 订单ID |
| user_id | BIGINT | - | NO | FK | - | 用户ID，外键 |
| total | DECIMAL | 10,2 | NO | - | 0.00 | 订单总价（美元） |
| purchased_at | DATETIME | - | NO | - | CURRENT_TIMESTAMP | 购买时间 |

**索引**：
- PRIMARY KEY: `id`
- INDEX: `user_id` (用于查询用户订单)
- INDEX: `purchased_at` (用于按时间排序)
- FOREIGN KEY: `user_id` REFERENCES `users(id)`

**约束**：
- `total` >= 0
- `purchased_at` 自动记录订单创建时间

**示例数据**：
```sql
INSERT INTO orders (user_id, total, purchased_at) VALUES
(1, 59.99, '2026-06-23 15:41:31'),
(1, 59.99, '2026-06-23 15:50:17'),
(2, 29.99, '2026-06-23 16:00:00');
```

**关联关系**：
- 通过`order_products`中间表关联多个商品

---

### 表5: order_products (订单商品关联表)

**表说明**：多对多关系中间表，连接订单和商品

| 字段名 | 数据类型 | 长度 | 是否为空 | 是否主键 | 默认值 | 注释 |
|--------|---------|------|---------|---------|-------|------|
| order_id | BIGINT | - | NO | PK | - | 订单ID，复合主键 |
| product_id | BIGINT | - | NO | PK | - | 商品ID，复合主键 |

**索引**：
- PRIMARY KEY: `(order_id, product_id)` - 复合主键
- FOREIGN KEY: `order_id` REFERENCES `orders(id)` ON DELETE CASCADE
- FOREIGN KEY: `product_id` REFERENCES `products(id)` ON DELETE RESTRICT

**约束**：
- **复合主键**：防止同一订单重复添加同一商品
- **级联删除**：删除订单时自动删除关联记录
- **限制删除**：不能删除已在订单中的商品

**示例数据**：
```sql
INSERT INTO order_products (order_id, product_id) VALUES
(1, 1),  -- 订单1包含商品1 (Elden Ring)
(2, 17), -- 订单2包含商品17 (Dark Souls III)
(3, 5),  -- 订单3包含商品5
(3, 8);  -- 订单3包含商品8
```

**业务说明**：
- 一个订单可以包含多个商品
- 一个商品可以出现在多个不同订单中
- 这是实现"个人游戏库"的核心表

## 三、数据库关系详解

### 关系1：User ─┬─ CartItem（一对多）

**关系类型**：一个用户可以有多个购物车项

**JPA映射**（在User实体中）：
```java
@OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
private List<CartItem> cartItems;
```

**SQL查询示例**：
```sql
-- 查询用户的购物车
SELECT ci.*, p.name, p.price, p.discount_percent
FROM cart_items ci
JOIN products p ON ci.product_id = p.id
WHERE ci.user_id = 1;
```

---

### 关系2：User ─┬─ Order（一对多）

**关系类型**：一个用户可以有多个订单

**JPA映射**（在User实体中）：
```java
@OneToMany(mappedBy = "user")
private List<Order> orders;
```

**SQL查询示例**：
```sql
-- 查询用户的所有订单
SELECT * FROM orders
WHERE user_id = 1
ORDER BY purchased_at DESC;
```

---

### 关系3：CartItem ─── Product（多对一）

**关系类型**：多个购物车项可以指向同一商品

**JPA映射**（在CartItem实体中）：
```java
@ManyToOne
@JoinColumn(name = "product_id", nullable = false)
private Product product;
```

---

### 关系4：Order ◄─► Product（多对多）

**关系类型**：一个订单包含多个商品，一个商品可以在多个订单中

**JPA映射**（在Order实体中）：
```java
@ManyToMany(fetch = FetchType.EAGER)
@JoinTable(
    name = "order_products",
    joinColumns = @JoinColumn(name = "order_id"),
    inverseJoinColumns = @JoinColumn(name = "product_id")
)
private List<Product> products;
```

**SQL查询示例**：
```sql
-- 查询订单包含的所有商品
SELECT p.* 
FROM products p
JOIN order_products op ON p.id = op.product_id
WHERE op.order_id = 1;

-- 查询用户购买的所有游戏（去重）
SELECT DISTINCT p.*
FROM products p
JOIN order_products op ON p.id = op.product_id
JOIN orders o ON op.order_id = o.id
WHERE o.user_id = 1;
```

## 四、重要SQL查询

### 查询1：获取用户购物车及总价

```sql
SELECT 
    ci.id,
    p.id AS product_id,
    p.name,
    p.price,
    p.discount_percent,
    p.price * (1 - p.discount_percent / 100.0) AS final_price
FROM cart_items ci
JOIN products p ON ci.product_id = p.id
WHERE ci.user_id = ?;

-- 计算总价
SELECT SUM(p.price * (1 - p.discount_percent / 100.0)) AS cart_total
FROM cart_items ci
JOIN products p ON ci.product_id = p.id
WHERE ci.user_id = ?;
```

---

### 查询2：获取用户已购买的所有游戏（个人库）

```sql
SELECT DISTINCT 
    p.id,
    p.name,
    p.header_image_url,
    p.price,
    p.developer,
    p.publisher
FROM products p
JOIN order_products op ON p.id = op.product_id
JOIN orders o ON op.order_id = o.id
WHERE o.user_id = ?
ORDER BY p.name;
```

---

### 查询3：获取用户订单历史

```sql
SELECT 
    o.id,
    o.total,
    o.purchased_at,
    COUNT(op.product_id) AS product_count
FROM orders o
LEFT JOIN order_products op ON o.id = op.order_id
WHERE o.user_id = ?
GROUP BY o.id
ORDER BY o.purchased_at DESC;
```

---

### 查询4：获取订单详情（包含商品列表）

```sql
SELECT 
    o.id AS order_id,
    o.total,
    o.purchased_at,
    p.id AS product_id,
    p.name AS product_name,
    p.header_image_url
FROM orders o
JOIN order_products op ON o.id = op.order_id
JOIN products p ON op.product_id = p.id
WHERE o.id = ?;
```

---

### 查询5：检查用户是否拥有某个游戏

```sql
SELECT EXISTS(
    SELECT 1
    FROM orders o
    JOIN order_products op ON o.id = op.order_id
    WHERE o.user_id = ? AND op.product_id = ?
) AS is_owned;
```

---

### 查询6：统计用户购买游戏数量

```sql
SELECT COUNT(DISTINCT op.product_id) AS total_games
FROM orders o
JOIN order_products op ON o.id = op.order_id
WHERE o.user_id = ?;
```

## 五、数据完整性约束

### 1. 主键约束（PRIMARY KEY）
- 所有表都有自增主键`id`
- `order_products`使用复合主键`(order_id, product_id)`

### 2. 唯一约束（UNIQUE）
- `users.username` - 用户名唯一
- `users.email` - 邮箱唯一
- `cart_items(user_id, product_id)` - 防止重复添加

### 3. 外键约束（FOREIGN KEY）
- `cart_items.user_id` → `users.id` (ON DELETE CASCADE)
- `cart_items.product_id` → `products.id` (ON DELETE CASCADE)
- `orders.user_id` → `users.id`
- `order_products.order_id` → `orders.id` (ON DELETE CASCADE)
- `order_products.product_id` → `products.id` (ON DELETE RESTRICT)

### 4. 非空约束（NOT NULL）
- 所有主键字段
- `users.username`, `users.password`, `users.email`
- `products.name`, `products.price`
- `orders.total`, `orders.purchased_at`

### 5. 检查约束（CHECK）
- `products.price >= 0`
- `products.discount_percent BETWEEN 0 AND 100`
- `orders.total >= 0`

## 六、数据库性能优化

### 索引策略

1. **主键索引**（自动创建）
   - 所有表的`id`字段

2. **唯一索引**
   - `users.username`
   - `users.email`
   - `cart_items(user_id, product_id)`

3. **外键索引**
   - `cart_items.user_id`
   - `cart_items.product_id`
   - `orders.user_id`
   - `order_products.order_id`
   - `order_products.product_id`

4. **查询优化索引**
   - `products.name` - 用于搜索
   - `products.featured` - 用于精选查询
   - `orders.purchased_at` - 用于按时间排序

### 查询优化建议

1. **避免N+1查询**
   - 使用`FetchType.EAGER`预加载关联数据
   - 或使用`@EntityGraph`

2. **使用分页**
   - 商品列表分页展示
   - 订单历史分页加载

3. **缓存策略**（未来扩展）
   - Redis缓存热门商品
   - Redis缓存用户Session
