# 系统功能及核心模块清单文档

## 一、系统概述

本系统旨在**为用户提供便捷的在线游戏购买、浏览和个人游戏库管理平台**，实现数字游戏商品的在线交易和用户购买记录管理。基于 **Spring Boot 3.2.0、Spring Security、JPA/Hibernate、MySQL 8.0、Thymeleaf** 等技术搭建，具备**高稳定性、强安全性、易扩展性、国际化支持**等特点，可满足**游戏爱好者、数字商品购买用户、个人收藏管理用户**在**在线游戏零售、数字商品交易、个人游戏库管理**的需求。

## 三、功能模块详细说明

### 模块1：用户系统模块

**模块概述**：负责用户的注册、登录、认证和会话管理，确保系统安全性。

**核心功能**：

1. **用户注册**
   - 输入：用户名、邮箱、密码
   - 验证：用户名唯一性、邮箱格式、密码强度
   - 处理：BCrypt加密存储密码
   - 输出：注册成功跳转登录页

2. **用户登录**
   - 输入：用户名、密码
   - 验证：Spring Security认证
   - 处理：创建Session、设置认证信息
   - 输出：登录成功跳转个人库

3. **用户退出**
   - 处理：清除Session、销毁认证信息
   - 输出：跳转登录页

**技术实现**：
- Spring Security配置认证流程
- CustomUserDetailsService加载用户
- BCryptPasswordEncoder密码加密
- Session管理（30分钟超时）

---

### 模块2：商品管理模块

**模块概述**：管理游戏商品的展示、查询和详情查看，为用户提供丰富的商品信息。

**核心功能**：

1. **商品浏览**
   - 展示所有游戏列表
   - 支持精选游戏推荐
   - 显示价格、折扣、评价

2. **商品详情**
   - 游戏名称、描述
   - 价格、折扣百分比、最终价格
   - 开发商、发行商
   - 游戏标签
   - 用户评价摘要
   - 是否已拥有标识

3. **商品搜索**
   - 按名称模糊搜索
   - 按标签筛选
   - 按价格区间筛选

**数据来源**：
- 数据库products表
- 130+款游戏数据
- 支持动态添加

**技术实现**：
- StoreController处理请求
- StoreService业务逻辑
- ProductRepository数据访问
- Thymeleaf模板渲染

---

### 模块3：购物车模块

**模块概述**：管理用户的临时购买意向，支持商品添加、移除和总价计算。

**核心功能**：

1. **添加到购物车**
   - 验证用户登录状态
   - 检查商品是否已在购物车
   - 创建购物车项关联
   
2. **查看购物车**
   - 展示所有购物车商品
   - 计算每个商品折后价
   - 计算总价
   - 显示商品数量

3. **移除商品**
   - 根据商品ID删除购物车项
   - 自动更新总价

**业务规则**：
- 同一商品不可重复添加
- 购物车与用户一对多关联
- 结账后自动清空购物车

**技术实现**：
- CartItem实体关联User和Product
- 唯一约束防止重复添加
- Stream API计算总价
- @Transactional事务保护

---

### 模块4：订单管理模块

**模块概述**：处理用户的购买流程，生成订单记录，管理订单历史。

**核心功能**：

1. **创建订单（结账）**
   - **业务流程**：
     ```
     1. 验证购物车非空
     2. 计算购物车总价
     3. 创建Order实体
     4. 关联订单商品（多对多）
     5. 保存订单到数据库
     6. 清空购物车
     7. 返回订单确认页
     ```
   
2. **订单历史查询**
   - 按时间倒序展示所有订单
   - 显示订单ID、购买时间、总价
   
3. **订单详情**
   - 订单包含的所有游戏
   - 每个游戏的价格
   - 订单总价
   - 购买时间

**数据关系**：
- Order与User：多对一
- Order与Product：多对多（通过order_products中间表）

**技术实现**：
- @Transactional确保数据一致性
- @ManyToMany(fetch = EAGER)避免懒加载问题
- OrderRepository.findByUser()查询用户订单

---

### 模块5：个人游戏库模块

**模块概述**：展示用户已购买的所有游戏，提供个人游戏收藏管理功能。

**核心功能**：

1. **查看个人库**
   - 展示所有已购买游戏
   - 显示游戏封面、名称
   - 标记"已拥有"

2. **拥有状态检测**
   - 在商店页面自动标记已拥有游戏
   - 已拥有游戏显示"✓ 已在库中"
   - 未拥有游戏显示"添加到购物车"

**数据来源**：
- 从orders表关联查询
- 通过order_products获取所有购买的product_id
- 去重处理（一个游戏只显示一次）

**技术实现**：
- `StoreService.getPurchasedProducts()`: 获取已购游戏列表
- `StoreService.getOwnedProductIds()`: 获取已拥有游戏ID集合
- 前端条件判断显示不同按钮

## 四、核心业务流程

### 业务流程1：用户购买游戏流程

```
用户 → 浏览商店 → 查看游戏详情 → 添加到购物车 → 查看购物车 
  → 点击结账 → 系统创建订单 → 清空购物车 → 游戏加入个人库
```

**详细步骤**：

1. **用户浏览商店** (`/store`)
   - 显示所有游戏列表
   - 已拥有游戏显示"✓ 已在库中"

2. **查看游戏详情** (`/store/game/{id}`)
   - 显示游戏完整信息
   - 判断是否已拥有
   - 显示对应按钮（添加购物车/已拥有）

3. **添加到购物车** (`POST /cart/add`)
   - 验证用户登录
   - 检查是否重复
   - 保存购物车项

4. **查看购物车** (`/cart`)
   - 显示所有购物车商品
   - 计算总价（含折扣）
   - 提供移除功能

5. **点击结账** (`POST /checkout`)
   - 开启事务
   - 验证购物车非空
   - 创建Order实体
   - 关联Products
   - 保存订单
   - 清空购物车
   - 提交事务

6. **订单确认** (`/checkout-success`)
   - 显示订单详情
   - 提供继续购物链接

7. **个人库展示** (`/dashboard`)
   - 查询用户所有订单
   - 提取所有已购游戏
   - 展示在个人库页面

---

### 业务流程2：数据库关系流转

```
User表
  ├─ 1:N → CartItem表 (购物车)
  │           └─ N:1 → Product表
  │
  └─ 1:N → Order表 (订单)
              ├─ M:N → Product表 (通过order_products中间表)
              └─ 订单创建后，Products自动加入用户游戏库
```

**关系说明**：

1. **User → CartItem**：一对多
   - 一个用户可以有多个购物车项
   - 每个购物车项属于一个用户

2. **CartItem → Product**：多对一
   - 多个购物车项可以指向同一个商品
   - 每个购物车项对应一个商品

3. **User → Order**：一对多
   - 一个用户可以有多个订单
   - 每个订单属于一个用户

4. **Order ↔ Product**：多对多
   - 一个订单可以包含多个商品
   - 一个商品可以出现在多个订单中
   - 通过`order_products`中间表实现

---

### 业务流程3：认证授权流程

```
1. 用户访问受保护URL (/dashboard, /cart, /checkout)
   ↓
2. Spring Security FilterChain拦截
   ↓
3. 检查是否已认证
   ↓
   [未认证] → 重定向到 /login
   ↓
4. 用户输入用户名和密码
   ↓
5. CustomUserDetailsService.loadUserByUsername()
   ↓
6. 从数据库加载用户信息
   ↓
7. BCryptPasswordEncoder验证密码
   ↓
   [验证失败] → 返回登录页，显示错误
   ↓
   [验证成功] → 创建Authentication对象
   ↓
8. 存入SecurityContext和Session
   ↓
9. 重定向到原始目标URL或默认的 /dashboard
   ↓
10. 后续请求自动携带Session，无需重复登录
```
